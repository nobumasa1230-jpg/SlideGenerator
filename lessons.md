## 2026-04-06 初期セットアップ

### Gemini APIモデル名
- **問題**: `gemini-2.0-flash-exp` が404で見つからない
- **原因**: モデル名が変更されていた
- **対策**: 画像生成には `gemini-3-pro-image-preview` を使用する

### GAS getTextStyle エラー
- **問題**: `textRange.getTextStyle is not a function` エラー
- **原因**: `setTextStyle` 関数で `getTextStyle()` を二重呼びしていた
- **対策**: 引数として `TextStyle` オブジェクト自体を渡す（`getText().getTextStyle()` の結果）

### GAS POST リダイレクト問題
- **問題**: curlのPOSTがGAS Web Appのリダイレクトで405エラー
- **原因**: GAS Web AppはPOST時にリダイレクトするが、curlはリダイレクト後GETに変わる
- **対策**: Python `requests` ライブラリで送信する（`allow_redirects=True` でリダイレクト追従）

### 画像背景色
- **問題**: テンプレートは白背景なのにダーク背景で生成された
- **原因**: `generate_image.py` のシステムプロンプトがダークブルー背景を指定していた
- **対策**: プロンプトで背景色を明示的に指定する（`CRITICAL: WHITE background (#FFFFFF)`）

### 画像アスペクト比
- **問題**: 正方形画像がGoogleスライド（16:9）に合わない
- **原因**: 初期設定が1:1正方形だった
- **対策**: 16:9ワイドスクリーンを明示的に指定する

## 2026-04-06 画像生成の品質問題（ユーザーフィードバック）

### 表紙のバリエーション不足
- **問題**: 表紙がシンプルになりすぎる（ダーク背景+テキストだけ）
- **原因**: 表紙テンプレートのバリエーションが1パターンしかない
- **対策**: テンプレートフォルダに表紙のバリエーションを複数用意する。表紙生成時はデザイン要素（アイコン、図形、帯ラベル、装飾線など）を必ず含めるようプロンプトに指示する

### テキストの改行位置が不自然
- **問題**: 日本語テキストの改行タイミングが読みづらい（単語の途中で切れるなど）
- **原因**: Geminiが自動的に改行位置を決めるため、日本語の文節を考慮しない
- **対策**: プロンプトでテキストの改行位置を明示的に指定する。例：「『AIで組織化してみた』動画が\n想定以上に再生された」のように、読みやすい区切りで改行を指示する

### 吹き出しの形状ミス
- **問題**: 吹き出しの突起（しっぽ）が2つになってしまう
- **原因**: Geminiの画像生成で吹き出し形状が不安定
- **対策**: プロンプトに「Each speech bubble must have exactly ONE tail/pointer」と明示する

### 文字間の不自然なスペース
- **問題**: 「想定以上」の「以」と「上」の間に不自然な半角スペースが入る
- **原因**: Geminiの日本語テキストレンダリングの不安定さ
- **対策**: プロンプトに「Japanese text must have NO extra spaces between characters. All characters must be evenly spaced with consistent kerning」と明示する

### テキスト内容の誤り
- **問題**: 「人に投げるだけで」→ 正しくは「AIに投げるだけで」。元メモの意味と異なるテキストが生成された
- **原因**: directorの構成案で「人に投げるだけで」と記載していたが、元メモの文脈では「AIに投げるだけで」が正しい
- **対策**: directorは元メモの表現をそのまま使うことを原則とする。意味を変えるような言い換えをしない。designerもプロンプトに含めるテキストは構成案の「」内を正確にコピーする。evaluatorは元メモとの照合でテキスト正確性を検証する
