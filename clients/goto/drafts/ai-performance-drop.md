# 構成案: AIでパフォーマンスが落ちた話 — 効率化という名の罠

## 概要
- 想定時間: 15〜20分
- 総スライド数: 20枚
- 画像生成必要数: 5枚（表紙、エネルギー分散図、サンクコスト比較、3つの委任フロー、まとめ宣言）

## スライド構成

### スライド1: 表紙
- **概念**: タイトル／表紙
- **タイプ**: image_full（画像生成）
- **内容**:
  - タイトル: 「AIでパフォーマンスが落ちた話」
  - サブタイトル: 「効率化という名の罠から抜け出すために」
- **デザイン指示**: ダークトーン背景。AIの光と影を表現するビジュアル。タイトルを大きく中央配置

### スライド2: 共感 — AIへの関心の高まり
- **概念**: 問題提起／警告
- **タイプ**: text
- **内容**:
  - タイトル: 「AIで組織化してみた」動画が想定以上に再生された
  - 本文: AIに興味を持つ人は確実に増えている。「すごい」と感じる人、「ついていけない」と戸惑う人、「やらないとまずい」と焦る人。あなたはどれですか？

### スライド3: AIの華やかな世界
- **概念**: 箇条書き（並列）
- **タイプ**: bullets
- **内容**:
  - タイトル: SNSで流れてくるAIの成功談
  - 項目:
    - 「放置するだけでこんなことができる」
    - 「人に投げるだけでここまでやってくれる」
    - 「AIチームが勝手に回してくれている」
    - 「画面を勝手に操作してくれる」

### スライド4: 自己紹介 — AI歴と検証実績
- **概念**: ワンメッセージ
- **タイプ**: text
- **内容**:
  - タイトル: 1年半以上、AIを本気で使い込んできた
  - 本文: Claude Chatは登場直後から使用。Claude Codeもここ数ヶ月は本腰を入れて検証。専門家に意見を聞きながら、自分の仕事・クライアントワークへの活用を重ねてきた。その中で構造的に見えてきたことを共有する。

### スライド5: 衝撃の結論
- **概念**: 大きな宣言
- **タイプ**: text
- **内容**:
  - タイトル: 結論：AIを使って、パフォーマンスは「下がった」
  - 本文: 振り返ると「ずっとAIで遊んでいただけ」という感覚が拭えない。成果を生んだか？自動化できたか？継続的に活用できたか？どれも答えは「NO」に近い。

### スライド6: セクション — 第一の罠
- **概念**: セクション区切り
- **タイプ**: section
- **内容**:
  - number: 1
  - タイトル: 終わりのない沼
  - サブタイトル: なぜAIに時間を吸い取られるのか

### スライド7: 終わりのない沼
- **概念**: 対比（2要素）
- **タイプ**: comparison
- **内容**:
  - タイトル: 2〜3時間の仕事に72時間かかる現実
  - option_a_title: 自分でやった場合
  - option_a:
    - 所要時間：2〜3時間
    - やるべきことが明確
    - 完了の基準がはっきりしている
  - option_b_title: AIで取り組んだ場合
    - option_b:
    - 所要時間：72時間（3日）
    - 「もっとこうした方がいい」が止まらない
    - AIが進化し続けるので終わりがない

### スライド8: セクション — 第二の罠
- **概念**: セクション区切り
- **タイプ**: section
- **内容**:
  - number: 2
  - タイトル: エネルギーが10分割される
  - サブタイトル: できることが多すぎるという落とし穴

### スライド9: AIでできることの広がり
- **概念**: 箇条書き（並列）
- **タイプ**: bullets
- **内容**:
  - タイトル: AIでできること、多すぎ問題
  - 項目:
    - 文章生成・画像生成・動画編集
    - LP制作・パワーポイント作成
    - スプレッドシート・Excel処理
    - GASによる自動化・Slack連携
    - 複数AIの並行稼働

### スライド10: エネルギー分散の図解
- **概念**: 対比（2要素）
- **タイプ**: image_full（画像生成）
- **内容**:
  - タイトル: 100%の集中 vs 10%ずつの分散
  - 左側：1つの仕事に100%のパワーを注ぐ図
  - 右側：10個の仕事に10%ずつ分散する図
- **デザイン指示**: 左右比較のインフォグラフィック。左は太い1本の矢印、右は細い10本の矢印。集中と分散のコントラストを視覚的に表現

### スライド11: 効率化していいもの、ダメなもの
- **概念**: 対比（2要素）
- **タイプ**: two_column
- **内容**:
  - タイトル: AIに任せていい仕事と、任せてはいけない仕事
  - left_title: 任せていい（＝効率化）
  - left:
    - スプレッドシートの関数入力
    - データ整理・フォーマット変換
    - 正誤が明確に判定できる作業
  - right_title: 任せてはいけない（＝手抜き）
  - right:
    - 自分の思い・価値観の表現
    - 情報発信の核となるメッセージ
    - 商品への熱量を伝える部分

### スライド12: 達成感の喪失
- **概念**: ワンメッセージ
- **タイプ**: text
- **内容**:
  - タイトル: 達成感がなくなると、仕事は続かない
  - 本文: 手抜きになるとアウトプットへのコミットメントが薄れる。仕事がただの作業になる。機械がやって終わってお金だけ稼げる — そんな仕事で本当にいいのか？実りのない仕事は続かない。

### スライド13: セクション — 第三の罠
- **概念**: セクション区切り
- **タイプ**: section
- **内容**:
  - number: 3
  - タイトル: サンクコストという「粘る力」が消える
  - サブタイトル: 時間をかけたものは簡単に手放せない、その力

### スライド14: サンクコストの両面
- **概念**: 対比（2要素）
- **タイプ**: image_full（画像生成）
- **内容**:
  - タイトル: 10時間かけたLP vs AIで5分のLP
  - 左：10時間かけて書いたLP → 「ここまでやったんだから絶対やり切る」→ 粘りが生まれる → 売れる傾向
  - 右：AIで5分で書いたLP → 数字が出なければ「じゃあ次」→ 20回テストで2ヶ月消費 → 売上ゼロのリスク
- **デザイン指示**: 左右比較のインフォグラフィック。左は上昇する矢印と炎のモチーフ、右は散らばる矢印。粘りの力と散漫さのコントラスト

### スライド15: 熱量が結果を左右する
- **概念**: ワンメッセージ
- **タイプ**: text
- **内容**:
  - タイトル: 自分の時間とエネルギーが乗っていないものは売れない
  - 本文: 自分が考え抜いて「これだ」と信じ切ったものは売れる。「こんな感じでいっか」で出したものは面白いほど売れない。商品の熱量がお客さんに届くかどうか、それが全て。

### スライド16: 3つの委任
- **概念**: フロー図
- **タイプ**: image_full（画像生成）
- **内容**:
  - タイトル: AIへの3つの委任 — どこまで渡すかが生命線
  - ステップ1: 実行の委任（やらせる）→ OK
  - ステップ2: 思考の委任（考えてもらう）→ 注意
  - ステップ3: 権限の委任（決めさせる）→ 危険
  - 補足：権限まで渡すと「顧客が見えなくなる」
- **デザイン指示**: 3段階のフロー図。緑→黄→赤のグラデーションで危険度を表現。最下段に「顧客不在」の警告を配置

### スライド17: 顧客不在の二極化
- **概念**: 対比（2要素）
- **タイプ**: comparison
- **内容**:
  - タイトル: これから起きる二極化
  - option_a_title: AIに飛びついた人
  - option_a:
    - 全てをAIに任せる
    - 顧客の顔が見えなくなる
    - 市場とズレていく
  - option_b_title: 顧客と向き合い続けた人
  - option_b:
    - 非効率でも地道に対話する
    - 顧客の悩み・欲求を肌で感じる
    - ビジネスの土台がどんどん強くなる

### スライド18: AIコンテンツの致命的リスク
- **概念**: 問題提起／警告
- **タイプ**: text
- **内容**:
  - タイトル: 「AIで書いた」とバレた瞬間、読まれなくなる
  - 本文: 誰かが時間とエネルギーをかけて書いたものだから、人はちゃんと読む。AIで書いたと分かった瞬間、冷める。使うなら絶対にバレないように。中途半端にAIで稼ごうとすると、自分のブランドが崩れていく。

### スライド19: 結論 — 3つの指針
- **概念**: 箇条書き（順序あり）
- **タイプ**: process
- **内容**:
  - タイトル: AIと付き合うための3つの指針
  - steps:
    - title: 線引きを決める
      description: 「AIに任せる領域」と「自分でやる領域」を事前に明確にしておく
    - title: 顧客との時間を守る
      description: 効率化で浮いた時間を、顧客を理解し関係性を深めることに使う
    - title: 粘る力を手放さない
      description: 効率化という名の手抜きに逃げず、自分の時間とエネルギーを注ぎ続ける

### スライド20: まとめ宣言
- **概念**: 大きな宣言
- **タイプ**: image_full（画像生成）
- **内容**:
  - メインメッセージ: 「AIに使われる側ではなく、AIを使いこなす側でいよう」
  - サブメッセージ: ビジネスの中心は、どこまで行ってもお客さん。効率化は手段であって、目的ではない。
- **デザイン指示**: ダークトーン背景にメインメッセージを大きく中央配置。力強いタイポグラフィ。サブメッセージを下部に小さく配置
